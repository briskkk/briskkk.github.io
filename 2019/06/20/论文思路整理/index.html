论文思路整理

发表于 2019-06-20 | 次阅读

正好要中期了，试着总结一下关于SDN，NFV，SFC。想到什么就说什么，做一下工作总结的归纳。关于中期报告书的编写，可以从中归纳出逻辑。

6.25

  1. SFC（service function chaining):服务功能链，一系列业务功能的有序组合，引导流量按序依次通过这些功能组件。

    • 传统方案的弊端：
      • 手动部署每个业务功能，单独部署特殊的设备提供各个业务功能。
      • 维护工作量大，业务缺乏灵活性
      • 可扩展性，当我们需要修改链，例如链中添加/删除业务功能时，需要手工操作，费时费力，容易出错

  2. 基于SDN/NFV的SFC：

    1. 业务功能被VNF取代，VNF动态创建，灵活扩展，更易部署，更有效利用资源，降低OPEX和CAPEX。
    2. 通过集中控制，控制器可以自动生成业务员功能链的流表

  3. 实现SFC的方式：

    1. OPNFV SFC：让ONOS支持NFV场景，让ONOS支持L2，L3，业务功能链特性，网络功能均是虚拟的（L2 switch，L3 router，classifier均是OVS虚拟的），业务转发器，DPI，防火墙均属于VNF。

      OPNFV即包含：OpenStack，ONOS，OV

      关于OPNFV的部署，可以在接下来的文档中介绍部署和实践。

      opnfv/fuel

    2. 查阅了相关资料，发现ODL控制器也有相关的SFC的usecase。入手应该更简单一下。

      • 基于ODL SFC demo环境搭建：
          1. 基于ODL SFC demo的环境搭建
          2. 基于ODL SFC demo的环境搭建

  4. SFC的流程：

    1. 创建服务功能链。从源主机发出的报文先到达分类器classifier，报文满足SFC过滤条件，classifier将其发送给FW，再经过DPI，最后发送给目的主机。classifier的另一个任务，就是向报文添加NSH报头，经过最后一个VNF时，再剥去NSH头部，将原始报文发往目的主机。
      • 可以配置防火墙，进行更细粒度的控制报文的进出
    2. 修改服务功能链。加入家长监控功能。

  5. Github上相关的项目总结：（可以学习和采纳）

    • OpenStackSanDiego/SecurityServiceCHains:描述创建安全相关功能链的脚本
    • ligato/sfc-controller:有内容描述SFC的拓扑和usecase
    • fortinet-solutions-cse/sfc-fgt:提供OpenStack/OPNFV/纯SFC的不同场景的解决方案，将Fortinet（一个防火墙吧）部署为VNF的方案。
    • javibr/OpenStack-And-SFC:提供使用Devstack安装部署OpenStack的配置文件以及部署SFC场景的脚本。由于是两年前的文档，可以借鉴参考部署SFC场景的脚本的编写。

6.24

关于使用机器学习建模：

学习能力（调参）

需要考虑批量大小，学习速率，学习速率计划，适应学习速率

模型，激活函数，节点数量，层的数量

如果出现梯度消失/爆炸，需要考虑批处理规范化，分层培训，迁移学习

泛化能力

需要考虑：权重正则化，增加噪声，提前停止，权重约束，活动规范化，随机失活

预测能力（减小预测方差）

需要考虑：模型均衡，堆栈集成，权重均衡

6.23

有三个问题：为什么我们需要SFC？怎么做？你的项目与网络的关系？

  1. NFV将网络服务部署为VM，虚拟机可认为是一个SFI

  2. SDN将SFC需要的网络控制功能下发

  3. SFC是一种实现Networking Service的标准方式，借助SDN和NFV能更好的实现SFC

  4. 用户对L4-L7的服务多种多样，如果都由SDN来实现，将大大增加SDN的复杂性，借助NFV/SFC，能够灵活实现Networking Service。

  5. SFC实现负载均衡：创建一组Service Function，实现相同的功能，前面加上负载均衡器可以让流量从一个通过。

    流量不是不需要steering，只是LB软硬件做了steering，SFC可以通过OpenFlow流表实现简单的LB

找到了关于SFC草案文档

简单总结了一小部分：

  • 增值服务（VAS）的交付依赖于各种业务功能的（SFs）的调用

  • 流量通过一组嵌入业务功能的网元进行转发

  • 用途：
    1. 将部分流量引导到网元上，实现监控和计费
    2. 向数据中心服务器发送流量钱，对流量导向至负载均衡器，将流量负载分担到多个链路及网元上
    3. 使用防火钱对流量过滤，实现IDS/IPS
    4. 使用安全网关对流量加解密（SSL offloading function）
    5. 流量可以穿越不同网络支持不同地址族（IPv4/IPv6）
    6. 部分内部服务平台依赖于服务表示，通过专用服务功能，丰富报文（HTTP头增强）
    7. TCP加速
