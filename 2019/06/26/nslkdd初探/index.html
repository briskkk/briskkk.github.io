nslkdd初探

发表于 2019-06-26 | 次阅读

关于NSL-KDD的说明：

NSL-KDD Dateset

关于和KDD99数据集的比较：

  1. NSL-KDD数据集的训练集中不包含冗余记录，所以分类器不会偏向更频繁的记录
  2. NSL-KDD数据集的测试集中没有重复的记录，使得检测率更为准确
  3. 来自每个难度级别组的所选记录的数量与原始KDD数据集中的记录的百分比成反比。结果，不同机器学习方法的分类率在更宽的范围内变化，这使得对不同学习技术的准确评估更有效。
  4. 训练和测试中的记录数量设置是合理的，这使得在整套实验上运行实验成本低廉而无需随机选择一小部分。因此，不同研究工作的评估结果将是一致的和可比较的。

关于特征的描述：41个特征

  1. 每个网络连接的基本特征 9种

    1. 持续时间duration：每个网络连接的持续时间，连续类型，定义是从TCP连接以3次握手建立算起，到FIN/ACK连接结束为止的时间。若为UDP，则将每个UDP数据包作为一条连接，数据集中大量duration=0的情况，是因为该条连接持续时间不足1秒。
    2. Protocol_type协议类型：3种，TCP，UDP，ICMP
    3. Service目标主机的网络服务类型：70种
    4. Flag:连接正常或错误的状态，11种，表示该连接是否按照协议要求开始或完成。SF表示状态正常，其他10种都是error
    5. Src_bytes:从源主机到目标主机的数据的字节数
    6. dst_bytes:从目标主机到源主机的数据的字节数
    7. land:若连接来自/送达同一个主机的端口则为1，否则为0
    8. wrong_fragement:错误分段的数量，连续类型
    9. urgent‘:加急包的个数

  2. TCP连接的内容特征 13种

    主要针对U2R和R2L攻击，不想DoS攻击那样在数据记录中具有频繁序列模式，一般都是嵌入在数据包的数据负载里面，单一的数据包和正常没有什么区别，所以从数据内容里面抽取了部分可能反映入侵行为的内容特征。

    1. hot：访问系统敏感文件的目录的次数，例如访问系统目录，建立或执行程序

    2. num_failed_logins：登陆尝试失败的次数

    3. logged_in：成功登陆则为1

    4. num_compromised:compromised条件出现的次数

    5. root_shell：是否获得root shell权限

    6. su_attempted：出现su root命令则为1

    7. num_root：root用户访问次数

    8. Num_file_creations：文件创建操作的次数

    9. num_shells：使用shell命令的次数

    10. num_access_files：访问控制文件的次数

    11. num_outbound_cmds：一个FTP会话中出战连接的次数

    12. is_hot_loigin：登陆是否属于hot列表

    13. is_guest_login：是否是guest登录

  1. 基于时间的网络流量统计特征 9种

    网络攻击事件在时间上有很强的关联性，因此统计当前连接记录与之前一段时间内的连接记录之间存在的某些联系可以更好的反应连接之间的关系。可以分为两种

    • same host：只观察在过去两秒内与当前连接有相同目标主机的连接
    • same service：只观察在过去两秒内与当前连接有相同服务的连接
    1. count.：过去两秒内，与当前连接具有相同的目标主机的连接数
    2. srv_count. 过去两秒内，与当前连接具有相同服务的连接数
    3. serror_rate. 过去两秒内，在与当前连接具有相同目标主机的连接中，出现“SYN” 错误的连接的百分比
    4. srv_serror_rate. 过去两秒内，在与当前连接具有相同服务的连接中，出现“SYN” 错误的连接的百分比
    5. rerror_rate. 过去两秒内，在与当前连接具有相同目标主机的连接中，出现“REJ” 错误的连接的百分比
    6. srv_rerror_rate. 过去两秒内，在与当前连接具有相同服务的连接中，出现“REJ” 错误的连接的百分比
    7. same_srv_rate. 过去两秒内，在与当前连接具有相同目标主机的连接中，与当前连接具有相同服务的连接的百分比
    8. diff_srv_rate. 过去两秒内，在与当前连接具有相同目标主机的连接中，与当前连接具有不同服务的连接的百分比
    9. srv_diff_host_rate. 过去两秒内，在与当前连接具有相同服务的连接中，与当前连接具有不同目标主机的连接的百分比

    23、25、27、29、30这5个特征是 “same host” 特征，前提都是与当前连接具有相同目标主机的连接

    24、26、28、31这4个特征是 “same service” 特征，前提都是与当前连接具有相同服务的连接

  2. 基于主机的网络流量统计特征

    基于时间的流量统计只是在过去两秒的范围内统计与当前连接之间的关系，而在实际入侵中，有些 Probing攻击使用慢速攻击模式来扫描主机或端口，当它们扫描的频率大于2秒的时候，基于时间的统计方法就无法从数据中找到关联。所以Wenke Lee等按照目标主机进行分类，使用一个具有100个连接的时间窗，统计当前连接之前100个连接记录中与当前连接具有相同目标主机的统计信息。

    1. dst_host_count. 前100个连接中，与当前连接具有相同目标主机的连接数
    2. dst_host_srv_count. 前100个连接中，与当前连接具有相同目标主机相同服务的连接数
    3. dst_host_same_srv_rate. 前100个连接中，与当前连接具有相同目标主机相同服务的连接所占的百分比
    4. dst_host_diff_srv_rate. 前100个连接中，与当前连接具有相同目标主机不同服务的连接所占的百分比
    5. dst_host_same_src_port_rate. 前100个连接中，与当前连接具有相同目标主机相同源端口的连接所占的百分比
    6. dst_host_srv_diff_host_rate. 前100个连接中，与当前连接具有相同目标主机相同服务的连接中，与当前连接具有不同源主机的连接所占的百分比
    7. dst_host_serror_rate. 前100个连接中，与当前连接具有相同目标主机的连接中，出现SYN错误的连接所占的百分比
    8. dst_host_srv_serror_rate. 前100个连接中，与当前连接具有相同目标主机相同服务的连接中，出现SYN错误的连接所占的百分比
    9. dst_host_rerror_rate. 前100个连接中，与当前连接具有相同目标主机的连接中，出现REJ错误的连接所占的百分比
    10. dst_host_rerror_rate. 前100个连接中，与当前连接具有相同目标主机的连接中，出现REJ错误的连接所占的百分比

KDD99数据集是由DARPA98数据集经过数据挖掘和预处理后得到的。但KDD99与DARPA98并不是一一对应的，Wende Lee等人在处理原始连接数据时将部分重复数据去除，例如进行DoS攻击时产生大量相同的连接记录，就只取攻击过程中5分钟内的连接记录作为该攻击类型的数据集。同时，也会随机抽取正常(normal)数据连接作为正常数据集

攻击类型

  1. DOS

    拒绝服务，耗尽了受害者的资源，从而使其无法处理合法请求。例如：syn flooding，相关的特征有：source bytes，percentage of packets with errors

  2. Probing

    监视和其他探测攻击的目的是为了获取有关远程受害者的信息。例如：port scanning，相关的特征有：duration of connection, source bytes

  3. U2R

    对本地root权限的未授权访问，攻击者通过给该攻击类型霍思燕难过普通账户登陆受害者系统，并尝试通过利用受害者的某些漏洞来获得root权限。例如：number of file creations，相关的特征有：number of file creations, number of shell prompts invoked

  4. R2L

    从远程计算机进行未授权的访问，攻击者侵入远程计算机并获得受害计算机的本地访问权限，例如：password guessing，相关的特征：网络级别的特征（duration of connection, service requested）和主机级别的特征（number of failed login attempts）

分类方法

  • Decesion Tree：决策树将分类问题划分为子问题，它构建了一个决策树，该决策树又用于构建用于分类的模型
  • Neural Networks：由生物神经网络驱动的统计学习模型的集合，用于估计或近似依赖于大量训练数据的函数
  • Nearest Neighbor：此方法通过训练数据集保存所有类，并根据相似性度量对新类进行分类。

所讨论的方法都有显著特点和固有缺点。

  1. 决策树需要时间来构建树
  2. 当数据集增加时，KNN相当耗费时间
  3. 神经网络仅对数值数据起作用，这需要将数据集中的文本数据转换为数值。

合成方法如下：

  • C4.5
  • K-means
  • SVM
  • Apriori
  • PageRank
  • AdaBoost
  • K-Nearest Neighbor
  • Naive Bayes
